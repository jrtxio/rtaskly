## 实现标题栏显示数据库路径

### 需求分析
用户希望在 Taskly 应用的标题栏中除了显示应用名称外，还能显示当前连接的数据库路径，格式类似：`todo.db (C:\Users\jrtxio\AppData\Local\todo.db) - Taskly`

### 实现方案
1. **创建标题生成函数**：在 `main-frame%` 类中添加一个辅助函数，根据当前数据库连接状态生成标题文本
2. **更新窗口标题**：
   - 在数据库连接成功后更新标题
   - 在数据库断开连接后更新标题
   - 初始启动时根据是否有数据库路径更新标题
3. **标题格式**：
   - 连接数据库时：`文件名 (完整路径) - Taskly`
   - 未连接数据库时：`Taskly`

### 具体修改步骤
1. **在 `gui/main-frame.rkt` 中添加标题生成函数**：
   ```racket
   ;; 更新窗口标题
   (define (update-title)
     (define title
       (if (current-db-path)
           (let* ([db-path (string->path (current-db-path))]
                  [file-name (path->string (file-name-from-path db-path))])
             (format "~a (~a) - Taskly" file-name (current-db-path)))
           "Taskly"))
     (send this set-label title))
   ```

2. **在数据库连接函数中调用标题更新**：在 `connect-to-db` 函数末尾添加 `(update-title)`

3. **在数据库断开函数中调用标题更新**：在 `disconnect-database` 函数末尾添加 `(update-title)`

4. **在应用初始化时调用标题更新**：在 `init-app` 方法末尾添加 `(update-title)`

### 预期效果
- 首次启动应用且未连接数据库时：标题显示 `Taskly`
- 连接到数据库后：标题显示 `文件名 (完整路径) - Taskly`
- 断开数据库连接后：标题恢复为 `Taskly`
- 从命令行指定数据库路径启动时：标题显示对应数据库的文件名和路径

### 技术要点
- 使用 Racket GUI 的 `set-label` 方法更新窗口标题
- 利用 `current-db-path` 参数获取当前数据库路径
- 使用 `file-name-from-path` 函数提取文件名
- 使用 `format` 函数格式化标题字符串
